# Contributing

## Commit Format

Use Conventional Commits:
- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation-only change
- `chore:` maintenance

## Breaking Interface Changes

Any breaking interface change requires an RFC update and approval before merge.
Include the related RFC reference in your PR description.

## Pull Request Checklist

- [ ] Tests added/updated for behavior changes
- [ ] Documentation updated for interface or policy changes
- [ ] Security implications reviewed
- [ ] RFC updated if introducing breaking interface changes
